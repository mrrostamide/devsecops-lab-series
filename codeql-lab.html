Scenario 1 — Spotting an Unsafe C Function

Some C functions, like strcpy, copy data without checking whether it actually fits in the destination. If the input is larger than the buffer, it overflows — this can crash a program or, in the worst case, let an attacker run their own code.

CodeQL can scan an entire codebase and automatically flag every single call to a risky function like this, instead of a human having to read every line by hand.

The Vulnerable Code

vulnerable.c
#include <string.h> void copy_input(char *user_input) { char buffer[64]; strcpy(buffer, user_input); // no size check here }

Building the Query, Step by Step

STEP 1 — Plain English

"Look at every function call in the program. If the function being called is named strcpy, flag it."

STEP 2 — Pseudocode
FOR EACH function_call IN program: IF function_call.name == "strcpy": REPORT function_call AS "unsafe"
STEP 3 — Build the Real Query 0 / 4 in place

Drag the lines (using the ⠿ handle) into the correct order, or use the ▲ / ▼ buttons. The border turns green the instant a line is in its correct spot.

  • ⠿where call.getTarget().hasName("strcpy")
  • ⠿import cpp
  • ⠿select call, "Unsafe use of strcpy"
  • ⠿from FunctionCall call

Scenario 2 — Another Unsafe Function: gets

gets reads a line of text from the keyboard into a buffer — but unlike strcpy, it doesn't even take a size argument to (mis)use. It has absolutely no way to know how big the destination buffer is, so it will keep writing for as long as the user keeps typing.

This function is so dangerous that it was formally removed from the C language standard in 2011. Let's write a CodeQL query to catch it, using the same pattern from Scenario 1.

The Vulnerable Code

vulnerable2.c
#include <stdio.h> void read_input(void) { char buffer[64]; gets(buffer); // no size limit at all — always risky }

Building the Query, Step by Step

STEP 1 — Plain English

"Look at every function call in the program. If the function being called is named gets, flag it."

STEP 2 — Pseudocode
FOR EACH function_call IN program: IF function_call.name == "gets": REPORT function_call AS "unsafe"
STEP 3 — Build the Real Query 0 / 4 in place

Same idea as Scenario 1, but this time the query targets gets. Drag the lines into order, or use the ▲ / ▼ buttons.

  • ⠿select call, "Unsafe use of gets"
  • ⠿from FunctionCall call
  • ⠿import cpp
  • ⠿where call.getTarget().hasName("gets")
