What is a branch?

Back in Act 0, we saw the word "main" and promised to explain it here.

Plain words: A branch is a separate copy of the timeline, a safe space to try something without touching main until you're ready. If the idea doesn't work out, you can throw the branch away, main was never at risk.

See your branches, then make a new one

git branch

Make the change, on the branch

Edit index.html, add a line right below the heading:

<p>Made by the StackSafe team.</p>
git add index.html
git commit -m "Add team line"
You only typed one new line, so this can look like a mistake. It isn't. Most text files are supposed to end with an invisible "end of line" marker after the very last line. Your original file was missing that marker. When you added a new line, Git had to remove the old, incomplete last line and re-add it correctly, with the marker, plus your new line after it. That shows up as "1 deletion, 2 insertions" instead of just "1 insertion." Nothing is wrong, and nothing you typed was lost, this is just Git being precise about invisible formatting.
Predict: switch back to main right now, would this new line be there?
🔒 Answer the question above to continue.

Plain words: This is the whole point of branching, main doesn't know about this change yet.

Bring it back together

git checkout main
git merge add-location
Predict: what happens?
🔒 Answer the question above to continue.

Plain words: A merge means: take everything from one branch and fold it into another. Because Priya's branch and main never touched the same thing at the same time, there was nothing for Git to be confused about.

Check the result

Plain words: The merge worked, both lines now show.

git branch

Notice add-location still exists. Merging doesn't delete the branch. Many teams delete it afterward since its job is done, but that's optional.

Reference

Quick Glossary

Open this anytime, from any Act. Looking it up never affects your progress or locks.

Git

Saves versions of your project. Works offline.

GitHub

The website where Git projects are shared with a team.

Repository

A project folder Git is watching. Also called a "repo."

Codespaces

A ready-made computer in your browser. Nothing to install.

The terminal

Where you type commands and press Enter to run them.

Staging area

Where you collect exactly what you want to save next, using git add.

Commit

A permanent save point in Git's history. Made with git commit.

Fork

Your own personal copy of someone else's repository on GitHub.

Branch

A separate copy of the timeline, a safe space to try something without touching main.

Merge

Folding one branch's changes into another. Done with git merge.

.gitignore

Tells Git to never track certain files, but only from now on, not the past.

API key

A private password proving your program is allowed to use a service.

Rotate (a key)

Replacing an exposed credential with a new one. The real fix after a leak.
