B
What actually happened

Denver International Airport, automated baggage system

In 1990, Denver began building a new airport with one automated baggage system meant to serve all three of its concourses. The system was designed as a single connected piece. It was not split into smaller parts that were built and tested separately as each one was finished.

All three concourses were served by one connected system, with no separate sections built or tested on their own.

Partway through construction, the airlines using the airport asked for changes that had not been part of the original plan, such as different handling for oversized bags and separate tracks for damaged equipment carts. By this point the system's design had already been finalized, so making these changes meant reworking parts of the system that were already considered finished.

The full system was tested only as one complete whole. Each time it was run this way, problems appeared: carts collided with each other, conveyor belts jammed, and the software could not coordinate the number of carts moving through the system at once. The airport's opening was postponed repeatedly while the team tried to fix the whole system at the same time.

2
The airlines' changes were requested after the plan had already been finalized, so making them meant reworking parts of the system that were already considered finished. What does this tell you about the cost of changing a plan after it is locked in, compared to if the same change had been known from the very start?
