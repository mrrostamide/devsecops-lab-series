📖 Quick Reference — ATT&CK Tactics (click to expand — stays available on both scenarios)

Full framework: attack.mitre.org — Tactics run across the top of the matrix (the attacker's goal at each stage); Techniques and Sub-techniques are listed underneath each Tactic (the specific method used to achieve it).

Reconnaissance
Gathering information to plan an attack.
Resource Development
Building or acquiring infrastructure, accounts, or tools to use later.
Initial Access
Getting a first foothold inside the target environment.
Execution
Running attacker-controlled code on a system.
Persistence
Keeping access alive across reboots, logoffs, or other interruptions.
Privilege Escalation
Gaining higher-level permissions than initially obtained.
Defense Evasion
Avoiding detection by security tools and defenders.
Credential Access
Stealing account names, passwords, or keys.
Discovery
Learning about the systems, accounts, and network layout.
Lateral Movement
Moving from one system to another within the network.
Collection
Gathering the data the attacker is ultimately after.
Command and Control
Communicating with and directing compromised systems.
Exfiltration
Getting stolen data out of the network.
Impact
Manipulating, interrupting, or destroying systems or data.

These definitions describe the category only — matching a specific event to a Tactic or Technique still takes reasoning about what the attacker was doing at that moment.

Scenario Brief

The Contractor's Laptop

A mid-sized logistics firm's SOC receives an alert three days after a routine vendor onboarding. A contractor's laptop, freshly issued and connected to the corporate VPN, begins exhibiting unusual behavior. The following is pieced together from EDR telemetry, VPN logs, and email gateway records.

  1. 09:14Contractor receives an email titled "Updated Vendor Compliance Form" with an attached Excel file containing a macro. The macro is enabled by the user.
  2. 09:16A PowerShell process spawns from Excel and downloads a second-stage payload from a remote IP.
  3. 09:20A new scheduled task named WinUpdateCheck is created, set to run the payload at every system startup.
  4. 11:45The compromised host queries Active Directory for domain admin accounts using built-in net commands.
  5. 14:02An RDP session is initiated from the contractor's laptop to a finance department file server using credentials harvested earlier that day.
  6. 16:30Roughly 2GB of files from a "Payroll" share are compressed into a password-protected archive and staged in a temp directory.
  7. 17:10The archive is uploaded via HTTPS POST to a cloud storage domain not on the corporate allow-list.

Scenario Brief

The Forgotten Key

A junior developer at a startup is working late to finish a feature before a demo. To save time, they hardcode an AWS access key directly into a Python script so they can test against the cloud environment, then push the code to the team's GitHub repository — which, without realizing it, is set to public. The key is meant only for a low-privilege testing account, but it was accidentally granted broader permissions months ago and never fixed.

  1. 22:14Developer commits deploy_test.py containing the AWS access key and secret, and pushes to the public GitHub repo.
  2. 22:19An automated bot that continuously scans public GitHub repositories for exposed credentials finds the key.
  3. 22:31The stolen key is used to log into the AWS account via the command line from an unfamiliar location.
  4. 22:40The attacker lists all S3 buckets and EC2 instances associated with the account to see what's available.
  5. 23:02Several large EC2 instances are launched in a region the company doesn't normally use — later found to be for cryptocurrency mining.
  6. 09:15 (+1)Finance flags an unusual cloud billing spike alert; the team investigates and finds the new instances and the exposed key.
