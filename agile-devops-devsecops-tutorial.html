Part 1 of 5 - The Three Layers

What does each layer actually do?

Each layer addresses a different problem. They are not alternatives to each other - they stack.

Agile

Organize and prioritize work

A project management philosophy using short, repeating cycles called sprints. The team builds small pieces, gets feedback, and adapts.

  • Sprint planning and reviews
  • Backlog of prioritized tasks
  • Daily standups
  • Retrospectives after each sprint
  • User stories as requirements
DevOps

Automate delivery and operations

A set of practices and tools that automate the path from written code to running software. Developers and operations work as one team.

  • Continuous Integration (CI)
  • Continuous Deployment (CD)
  • Infrastructure as Code (IaC)
  • Automated testing pipelines
  • Monitoring and alerting
DevSecOps

Embed security everywhere

Security checks are automated and built into the CI/CD pipeline. Rather than a gate at the end, security is everyone's responsibility from day one.

  • SAST - static code analysis
  • DAST - dynamic app testing
  • Dependency vulnerability scanning
  • Secrets detection in code
  • Policy and compliance as code
Why they need each other: Agile creates the demand for frequent delivery - end of every sprint, something should ship. DevOps creates the technical capability to actually do that safely and automatically. Without DevOps, Agile sprints pile up as "done but not deployed" features. DevSecOps then ensures that speed does not create security debt: vulnerabilities caught in the pipeline cost 10-100x less to fix than those caught in production.
