Part 1 of 5 - Core Concepts

What does each layer do?

Select a topic to explore the concept, what problem it solves, and a real-world analogy.

Version Control (VCS)

Track every change to your code

A Version Control System records every modification to your codebase, who made it, and when. It lets multiple developers collaborate on the same code without overwriting each other's work.

  • Commits - snapshots of code at a point in time
  • Branches - parallel lines of development
  • Merging - combining branches back together
  • Pull requests - proposing and reviewing changes
  • Git is the dominant VCS; GitHub/GitLab host repositories
Analogy
Google Docs version history - but for code

Like Google Docs lets you see every edit ever made and revert to any version, Git does the same for code - but far more powerfully. Multiple people can work on separate "copies" (branches) simultaneously and merge them together without conflict.

Problem it solves "My colleague and I both edited the same file and overwrote each other's work." / "I deleted code that was working and can't get it back." / "Who changed this function and why?"
Infrastructure as Code (IaC)

Define your infrastructure in files

Instead of manually clicking through a cloud console to create servers, networks, and databases, IaC lets you describe all of that in configuration files - then apply them automatically and repeatedly.

  • Declarative - you describe the desired state, not the steps
  • Idempotent - run it 10 times, get the same result
  • Terraform provisions cloud resources (AWS, Azure, GCP)
  • Ansible configures software on existing servers
  • Infrastructure lives in Git, reviewed like any code
Analogy
A recipe for your infrastructure

A chef's recipe specifies exactly what ingredients and steps produce a dish - anyone can follow it and get the same result. IaC is a recipe for servers: you write it once, and anyone on any cloud can execute it to get an identical, reproducible environment.

Problem it solves "Setting up a new environment takes 3 days of manual work." / "Staging and production are slightly different and we don't know why." / "The engineer who set this up left and no one knows how to recreate it."
Containers

Package apps that run anywhere

A container bundles an application with everything it needs to run - code, runtime, libraries, settings - into a single portable unit. It behaves identically on a developer's laptop, a test server, and in production.

  • Docker creates and runs containers from images
  • Container image - the packaged, portable blueprint
  • Container registry - where images are stored (Docker Hub)
  • Kubernetes orchestrates many containers at scale
  • Microservices - one service per container, loosely coupled
Analogy
A standardized shipping container

Shipping containers transformed global trade by standardizing transport: regardless of contents, every container fits the same ship, crane, and truck. Docker containers do the same for software - they run on any machine regardless of what operating system or dependencies are installed underneath.

Problem it solves "It works on my machine but crashes on the server." / "Upgrading Python for one app broke three others." / "Deploying takes forever because every server needs different setup scripts."
Cloud Computing

On-demand infrastructure at scale

Cloud providers (AWS, Azure, GCP) offer computing, storage, networking, and databases as on-demand services over the internet. You pay for what you use and can scale up or down in seconds.

  • IaaS - raw VMs and networks (you manage the OS)
  • PaaS - managed platform (you just deploy code)
  • SaaS - fully managed software (e.g. Gmail, Salesforce)
  • Auto-scaling - add or remove capacity automatically
  • Multi-region - deploy globally in minutes
Analogy
Renting a hotel room vs buying a house

Buying a house means paying upfront, maintaining everything, and being stuck with fixed capacity. A hotel lets you book exactly the rooms you need for exactly as long as you need - and the hotel maintains the building. Cloud is the hotel model for computing: pay as you go, scale instantly, no hardware ownership.

Problem it solves "We bought $500k of servers that sit at 10% utilization most of the time." / "Traffic spiked 10x overnight and our servers crashed." / "It takes 6 weeks to provision new hardware for a project."

Part 2 of 5 - The Code to Cloud Journey

How all four layers connect

In a modern DevOps workflow, all four technologies form a continuous pipeline. Here is how a developer's code travels from their machine to a live application.

Version Control
Infrastructure as Code
Containers
Cloud
Reading this pipeline The developer writes code and commits it to a Git repository (VCS). That commit triggers CI/CD: Terraform provisions or updates any cloud infrastructure needed (IaC). Docker then builds a container image of the application (Containers) and pushes it to a registry. Kubernetes - running on cloud infrastructure - pulls the image and deploys it. The live application runs on cloud compute, exposed through a cloud load balancer. Each layer depends on the one before it.

Key tools by layer

Version Control tools

  • Git (the protocol)
  • GitHub (hosted repos + CI)
  • GitLab (self-hosted option)
  • Conventional Commits
  • Git Flow (branching strategy)

IaC tools

  • Terraform (cloud provisioning)
  • Ansible (configuration management)
  • AWS CloudFormation
  • Pulumi (code-first IaC)
  • Helm (K8s package manager)

Container tools

  • Docker (build and run)
  • Kubernetes (orchestration)
  • Docker Compose (local multi-container)
  • Podman (daemonless alternative)
  • containerd (runtime standard)

Cloud services

  • AWS EC2 / EKS / S3 / RDS
  • Azure AKS / Blob / Functions
  • GCP GKE / Cloud Run / BigQuery
  • Cloudflare (CDN + edge)
  • Vercel / Railway (PaaS)

Tutorial complete!

These four layers are the backbone of modern DevOps and DevSecOps infrastructure. In practice they work as one: code is versioned in Git, infrastructure is declared in Terraform, apps are packaged in Docker, and everything runs on cloud compute - all automated through a CI/CD pipeline.
